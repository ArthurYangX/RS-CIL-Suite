# HALM-CIL: 基于HSI锚定链接记忆的无样本多模态类增量学习

## 一句话总结

**存HSI原型当"钥匙"，用线性映射从HSI推导LiDAR，漂移补偿后用原型分类器做推理，不存任何原始样本。**

---

## 问题背景

### 场景
遥感图像分类中，我们有两种数据：
- **HSI（高光谱图像）**：每个像素有几十~几百个光谱波段，能区分材质
- **LiDAR（激光雷达）**：提供高度/地形信息，能区分高矮物体

模型需要**逐步学习新类别**（比如先学5类，再学5类，再学5类），但**不能存储旧数据**（隐私/存储限制）。

### 核心挑战
1. **灾难性遗忘**：学新类时，模型把旧类忘了
2. **表示漂移**：每学一个新任务，特征空间就会"漂移"，旧类的原型位置变了
3. **多模态漂移不同步**：HSI和LiDAR的特征漂移速度不一样（实验证实LiDAR漂移是HSI的2倍）
4. **分类器偏差**：不知道测试样本属于哪个任务时（TAg），分类器严重偏向最近学的类

---

## 方法概览

HALM-CIL 由三个组件构成：

```
┌──────────────────────────────────────────────────┐
│                   训练阶段                        │
│  和普通方法一样：交叉熵 + 知识蒸馏（logit KD）      │
│  没有额外的训练损失                                │
└──────────────────┬───────────────────────────────┘
                   │ 训练完一个任务后
                   ▼
┌──────────────────────────────────────────────────┐
│           组件A: HSI锚定链接记忆                    │
│  为每个类存储：                                     │
│    · HSI原型 (均值向量)                             │
│    · LiDAR原型 (均值向量)                           │
│  拟合一个线性映射 W: HSI原型 → LiDAR原型             │
│  HSI是"钥匙"，LiDAR是"可推导的值"                   │
└──────────────────┬───────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────┐
│           组件B: 逐模态漂移补偿                     │
│  用新任务数据估计特征空间的漂移：                      │
│    · HSI漂移：拟合仿射变换 A_h, b_h                 │
│    · LiDAR漂移：拟合仿射变换 A_l, b_l               │
│  把旧类原型"搬运"到新的特征空间中                     │
└──────────────────┬───────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────┐
│           组件C: 保真度加权原型分类器                 │
│  推理时用余弦相似度做分类：                           │
│    score = cos(HSI特征, HSI原型)                    │
│          + α × cos(LiDAR特征, LiDAR原型)            │
│  其中 α 由链接映射的残差决定：                        │
│    残差小 → α大 → LiDAR贡献多                       │
│    残差大 → α小 → 退化为仅用HSI                     │
└──────────────────────────────────────────────────┘
```

---

## 三个组件详解

### 组件A: HSI锚定链接记忆

**直觉**：光谱特征（HSI）比地形特征（LiDAR）更稳定。所以我们把HSI当作"主记忆"，LiDAR当作"附属记忆"。

**做法**：
1. 训练完一个任务后，对每个类 c 计算：
   - HSI原型：`m_h^c = 所有该类样本的HSI特征的均值`
   - LiDAR原型：`m_l^c = 所有该类样本的LiDAR特征的均值`

2. 用所有已知类的原型，拟合一个**岭回归映射** W：
   ```
   W = M_lidar × M_hsi^T × (M_hsi × M_hsi^T + λI)^{-1}
   ```
   这个 W 能从HSI原型"预测"对应的LiDAR原型。

**为什么叫"链接"**：HSI和LiDAR不是独立存储的，而是通过 W 关联起来的。HSI是钥匙，LiDAR可以从HSI推导出来。

**存储开销**：每个类只存 2×D 个浮点数（D是特征维度），加上一个 D×D 的映射矩阵 W。

---

### 组件B: 逐模态仿射漂移补偿

**问题**：学完新任务后，模型的特征提取器变了。旧类的原型是在"旧空间"里算的，现在在"新空间"里位置不对了。

**做法**：
1. 用**新任务的数据**同时过旧模型和新模型，得到成对的特征：
   ```
   z_old = 旧模型提取的HSI特征
   z_new = 新模型提取的HSI特征
   ```

2. 用**最小二乘法**拟合一个仿射变换：
   ```
   z_new ≈ A × z_old + b
   ```
   这是一个**封闭解**（不需要梯度下降），计算很快。

3. 用这个变换把所有旧类原型"搬运"到新空间：
   ```
   新原型 = A × 旧原型 + b
   ```

4. HSI和LiDAR**分别做**，因为它们漂移的方式不同。

**实验发现**：
- HSI的漂移量（TransErr ~35-50）比LiDAR（~60-80）小得多
- 这证实了HSI作为锚点的合理性
- **没有漂移补偿，方法完全崩溃**（从50%降到0%）

---

### 组件C: 保真度加权原型分类器

**问题**：普通的原型分类器对HSI和LiDAR一视同仁，但如果LiDAR原型不准确（链接映射误差大），盲目使用会降低准确率。

**做法**：
1. 计算链接映射的**残差**（衡量 W 预测 LiDAR 原型的准确程度）：
   ```
   r_c = ||实际LiDAR原型 - W × HSI原型||²
   ```

2. 将残差转换为**保真度权重**：
   ```
   α_c = exp(-γ × r_c)
   ```
   - 残差小 → α接近1 → LiDAR信息可信，正常使用
   - 残差大 → α接近0 → LiDAR不可信，退化为仅用HSI

3. 最终分类分数：
   ```
   score_c(x) = cos(x_hsi, 原型_hsi^c) + α_c × cos(x_lidar, 原型_lidar^c)
   预测类别 = argmax score_c(x)
   ```

**好处**：当链接映射失效时，方法会自动退化为HSI-only分类器，而不是给出错误结果。

## 与现有方法的区别

| 特点 | HyperKD | DCPRN | DPCR | GACL | **HALM-CIL** |
|------|---------|-------|------|------|-------------|
| 多模态(HSI+LiDAR) | 否(仅HSI) | 否 | 否 | 否 | **是** |
| 无样本 | 否(需要存样本) | 否 | 是 | 是 | **是** |
| 逐模态漂移补偿 | 否 | 否 | 单空间 | 无漂移 | **HSI/LiDAR分别补偿** |
| 跨模态链接记忆 | 否 | 否 | 否 | 否 | **HSI→LiDAR岭回归映射** |
| 分类器类型 | 神经网络头 | 原型+蒸馏 | 岭回归重建 | 解析闭式 | **保真度加权原型** |

**核心新颖点**：没有人在 HSI+LiDAR 无样本类增量学习中做过"链接记忆"——用一个模态的原型推导另一个模态。

## 当前实验结果

### Houston2013（15类，3个任务，每任务5类，30 epochs）

**消融实验**证明每个组件的必要性：

| 变体 | 链接记忆 | 漂移补偿 | 保真度门控 | TAg均值 | TAg遗忘 |
|------|---------|---------|----------|---------|---------|
| A1 | 否(仅融合原型) | 否 | 否 | 0.02% | 82.6% |
| A2 | 是 | **否** | 否 | 0.00% | 64.5% |
| A4 | 是 | **否** | 是 | 0.11% | 72.5% |
| A3 | 是 | **是** | 否 | ~50% | 28.4% |
| **A5** | **是** | **是** | **是** | **49.8%** | **27.8%** |

**关键结论**：
- 没有漂移补偿 → 灾难性崩溃（A1, A2, A4 全部 ≈ 0%）
- 漂移补偿是最关键的组件（A3 vs A2: 从0%到50%）
- 链接记忆 + 门控在有漂移补偿的基础上进一步改善（A5 vs A3）

### 当前瓶颈

- TAg 49.8% 和遗忘 27.8% 还不够好
- 根本原因：训练阶段骨干网络漂移太大（仿射变换偏离单位矩阵的范数 35-80）
- HALM 的后处理补偿只能部分修正
- 需要在训练阶段就减少漂移（更强的正则化/冻结策略），或集成到已有的 RFCIL 框架中
